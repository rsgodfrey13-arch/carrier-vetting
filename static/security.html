1. Security Philosophy

Carrier Shark is a neutral document facilitation platform used to manage broker–carrier contracts, insurance documents, and workflow data. Security is designed around protecting document integrity, account access, and platform reliability.

We design systems to minimize risk, limit access, and preserve document authenticity — without over-collecting or over-retaining data.

2. Infrastructure & Hosting

  • Cloud Hosting: DigitalOcean infrastructure.
  • Document Storage: DigitalOcean Spaces (S3-compatible object storage).
  • Databases: PostgreSQL (primary data) and Redis (caching and session support).
  • Processing Layer: Secure workflow processing components hosted in managed cloud environments.

Access to production systems is restricted and controlled through credential-based access controls.

3. Encryption

  • Encryption in transit using HTTPS/TLS.
  • Cloud provider–managed encryption for stored objects.
  • Controlled access to document storage endpoints.

Sensitive documents are not exposed publicly and are accessible only through authorized workflows.

4. Authentication & Access Control

  • Account-based access control.
  • Email-based verification codes for signature workflows.
  • Role-based access separation where applicable.
  • Administrative access limited to authorized personnel.

Users are responsible for safeguarding their account credentials.

5. Monitoring & Platform Integrity

  • Application monitoring and error tracking.
  • Logging of key authentication and workflow events.
  • Fraud and misuse detection mechanisms.
  • Rate limiting and abuse mitigation controls.

Monitoring is designed to protect document authenticity and prevent unauthorized system access.

6. Data Handling Practices

  • Documents stored only for operational and contractual purposes.
  • No sale of personal information.
  • No advertising tracking or behavioral profiling.
  • Public carrier profile pages display public regulatory data only.

Uploaded insurance certificates and executed agreements are not publicly displayed.

7. Incident Response

Carrier Shark maintains procedures for investigating and responding to suspected security incidents.

  • Internal escalation and containment procedures.
  • Investigation of scope and impact.
  • Notification to affected customers where legally required.
Security concerns may be reported to: security@carriershark.com

8. Shared Responsibility

Security is a shared responsibility. Customers are responsible for:

  • Maintaining secure internal processes.
  • Verifying document authenticity before reliance.
  • Managing authorized access within their organization.

Carrier Shark provides infrastructure and controls, but customers determine how documents are used within their operations.
