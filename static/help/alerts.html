Alerts & Monitoring

Configure email alerts and connect Carrier Shark to your internal systems with API and webhook delivery.

4–8 minutes Intermediate

Email Alerts

Email alerts let you choose exactly which carrier changes should trigger notifications. You can enable or disable alert categories, customize the fields you care about, and save preferences for your account.

Step 1 — Open Email Alerts
  • Go to My Account and open Email Alerts.
  • Use the alerts master switch to enable or disable account-level email delivery.
  • If the master switch is off, carrier alerts will not be delivered by email.
Step 2 — Choose what to monitor
  • Select the carrier fields that should trigger alerts.
  • Categories can include authority, safety, compliance, identity, address, fleet, crashes, and more.
  • Choose only the changes that matter to your operation to keep alerts high-signal.
Step 3 — Save your alert preferences
  • After updating your selections, click Save changes.
  • Your account will use those settings for email delivery going forward.
  • You can return anytime to adjust which changes generate alerts.
Best practice
Start with authority, safety, insurance, and compliance-related changes. You can expand into lower-priority fields later once your workflow is dialed in.

Carrier-Level Controls

In addition to account-wide email settings, Carrier Shark also lets you control email alert delivery at the carrier level. This is useful when some carriers should stay monitored without sending email notifications for every record.

Step 1 — Open a carrier profile
  • Open any carrier from My Carriers or search.
  • Use the email alerts toggle at the top of the carrier profile to control alert delivery for that carrier.
  • This lets you keep monitoring active while controlling where notifications are sent.
Step 2 — Combine account and carrier settings
  • Account settings control what kinds of changes qualify as alerts.
  • Carrier-level toggles help you decide which monitored carriers should send email notifications.
  • Together, these controls make email delivery flexible for larger carrier lists.

API Notifications

Carrier Shark API access is available for teams that want to bring alert data and carrier events into internal systems. API access is intended for operational workflows, integrations, and custom implementations.

Step 1 — Open API settings
  • Go to My Account and open API.
  • Your API page includes key access, integration settings, and a link to documentation.
  • Use this page as your starting point for technical setup.
Step 2 — Use API access for internal workflows
  • API access is best for teams that want to route Carrier Shark data into internal tools or operational dashboards.
  • Your integration can be tailored to your workflow, system architecture, and alert requirements.
  • For setup help or implementation questions, contact integrations@carriershark.com.
Documentation
API documentation is available from the API section. If you need implementation guidance, contact integrations@carriershark.com.

Webhook Setup

Webhooks are available for teams that want Carrier Shark to push alert events into internal systems in real time. This is a strong fit for TMS integrations, operations dashboards, internal automations, and custom workflows.

Step 1 — Add your webhook destination
  • Open the API section in My Account.
  • Use the webhook area to define the destination for real-time event delivery.
  • Webhook delivery can be aligned to your operational workflow and supported integrations.
Step 2 — Coordinate supported integrations
  • For production webhook setup, contact integrations@carriershark.com.
  • Carrier Shark can support integrations into your existing systems and workflows.
  • This ensures your webhook delivery is configured correctly for your use case.

Delivery Notes

Carrier Shark is designed to support both self-serve alerting and operational integrations. The right delivery method depends on how your team works and where alert data needs to go.

Choose the right delivery method
  • Use Email Alerts for fast human-readable updates.
  • Use the REST API when your team wants access to data inside internal systems.
  • Use Webhooks for real-time automated delivery into supported workflows.
Need integration help?
  • For API onboarding, webhook configuration, or integration planning, contact integrations@carriershark.com.
  • The Carrier Shark team can help align alert delivery with your operational workflow.
  • If you are not sure which method fits best, start with email alerts and expand from there.
